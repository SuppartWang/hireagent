# HireAgent 产品需求文档（PRD）

> 版本：v1.0  
> 更新日期：2026-04-14

---

## 1. 产品概述

### 1.1 一句话定位

**HireAgent** 是一个 AI Agent 配置共享市场（Agent Marketplace）。用户可以浏览、发布、评价社区创作的 Agent 配置，并一键导出到 Claude Desktop、OpenClaw 等工具中。

### 1.2 核心灵感

- **clawhub**：实现了技能共用，同样的技能无需重复搭建。
- **evomap**：实现了胶囊共用，同样的问题无需重复解决、消耗 Token。
- **openclaw**：多 Agent 协作效果显著，但缺少一个可以分享 Agent 配置的产品。

**HireAgent 正是为了填补这个空白而生** —— 让优秀的 Agent 配置像开源代码一样被社区发现、复用和迭代。

---

## 2. 目标用户

### 2.1 创作者（Creator）

- 调优了一个在特定场景表现优异的 Agent
- 希望将配置分享给社区，获得认可与积分
- 可能包括开发者、产品经理、咨询师、教育工作者

### 2.2 使用者（Consumer）

- 不想从零开始写 System Prompt
- 希望快速找到适合某个任务的 Agent 配置
- 需要一键导入到 Claude、OpenClaw 或其他支持自定义 Agent 的工具

---

## 3. 用户旅程

### 3.1 发现 Agent

1. 用户进入首页，看到精选推荐（Featured）和本周热门（Trending）。
2. 进入广场页，通过搜索、分类筛选、能力标签找到目标 Agent。
3. 查看详情页，阅读系统提示词、MCP 配置、用户评价。
4. 点击"立即使用"，选择导出格式（Claude Desktop / OpenClaw / YAML / Prompt）。

### 3.2 发布 Agent

1. 注册/登录后，进入发布页。
2. 按步骤填写：基本信息 → 系统提示词 → MCP 配置 → 能力标签 → 预览发布。
3. 保存草稿或发布上线。
4. 首次发布获得积分奖励；后续被使用、被推荐、获五星好评均可继续获得积分。

### 3.3 评价与互动

1. 使用者在 Agent 详情页提交评分和评论。
2. 五星好评会自动为创作者发放积分。
3. 其他用户可以标记某条评价"有用"，评价者也会获得少量积分。

## 4. 功能模块

### 4.1 智能体广场（Marketplace）

| 功能 | 说明 |
|------|------|
| 搜索 | 支持按名称、标签全文搜索（PostgreSQL tsvector） |
| 分类筛选 | 编程开发、写作创作、研究调查、数据分析、客户服务、教育培训、创意设计、效率工具、法律咨询、财务金融、其他 |
| 能力筛选 | 网络搜索、代码执行、图像生成、文件读取、数据库查询、发送邮件、日历访问、浏览器控制、长期记忆、多智能体 |
| 排序 | 综合排名 / 最新发布 / 最高评分 / 最多使用 / 本周热门 |
| 视图切换 | 网格视图 / 列表视图 |

### 4.2 智能体详情（Agent Detail）

- 展示 Agent 的头像、名称、描述、分类、版本
- 系统提示词（可展开/收起、一键复制）
- MCP 配置（JSON 格式化展示）
- 能力标签与语言支持
- 用户评价列表与提交表单
- 导出入口（弹出 ExportModal）
- 在线试用入口（跳转 TryAgent 页面，模拟对话风格预览）

### 4.3 创作者中心（Creator Hub）

- 发布新 Agent（5 步向导式表单）
- 编辑已有 Agent（自动回显数据、保存版本历史）
- 我的 Agent 列表（草稿 / 已发布 / 精选）
- 积分历史与徽章等级
- 我的收藏

### 4.4 导出中心（Export）

| 格式 | 用途 |
|------|------|
| Claude Desktop | 下载 `claude_desktop_config.json`，合并到 Claude Desktop 的 MCP 配置中 |
| OpenClaw | 下载包含 `agent.md` + `openclaw.json` 合并指南的 bundle，快速接入多 Agent 工作流 |
| Generic JSON | 通用 Agent 配置 JSON，可被其他工具解析 |
| YAML | 通用 YAML 格式 |
| Prompt Text | 纯文本系统提示词，直接复制到任意 AI 工具的 System Prompt 设置中 |

每次导出都会记录一次 "hire" 事件，用于计算使用量和创作者积分。

### 4.5 管理员后台（Admin）

- 推荐/取消推荐 Agent 到首页 Featured
- 手动触发排名重新计算
- 查看平台统计数据（已发布 Agent 数、用户数、总使用量、总评价数）

## 5. 排序与奖励机制

### 5.1 排名分算法（Ranking Score）

定时任务每 6 小时自动重新计算所有已发布 Agent 的排名分：

```
Ranking Score = 0.4 × 评分归一化
              + 0.3 × 使用度
              + 0.2 × 质量分
              + 0.1 × 新鲜度
```

- **评分归一化**：`(avg_rating - 1) / 4`，范围 [0, 1]
- **使用度**：`log(hire_count + 1) / log(global_max_hires + 1)`，对数压缩防止头部垄断
- **质量分**：根据系统提示词长度、MCP 配置完整性、能力数量、描述长度、头像/演示视频、标签数量自动计算，范围 [0, 1]
- **新鲜度**：`exp(-0.023 × days_since_publish)`，新发布 Agent 获得一定加成

### 5.2 质量分细则

| 维度 | 条件 | 得分 |
|------|------|------|
| 系统提示词 | 长度 > 200 字符 | +0.30 |
| MCP 配置 | 包含至少 1 个 mcpServer | +0.25 |
| 能力数量 | ≥ 2 个 | +0.15 |
| 描述长度 | > 100 字符 | +0.10 |
| 头像 | 已上传头像 URL | +0.05 |
| 演示视频 | 已上传视频 URL | +0.10 |
| 标签数量 | ≥ 3 个 | +0.05 |
| **上限** | — | **1.00** |

### 5.3 积分规则（Point Rules）

| 行为 | 积分 | 触发条件 |
|------|------|----------|
| 智能体被使用 | +10 | 用户执行任意格式的导出 |
| 获得五星好评 | +50 | 有用户提交 5 星评价 |
| 被推荐到首页 | +100 | 管理员将 Agent 标记为 Featured |
| 首次发布 | +25 | 创作者第一次发布 Agent（从草稿变为 published） |
| 评价被标记有用 | +5 | 其他用户点击某条评价的"有用" |

### 5.4 徽章等级（Badge Tiers）

| 等级 | 积分范围 |
|------|----------|
| 新星 | 0 – 99 |
| 成长者 | 100 – 499 |
| 精英 | 500 – 1999 |
| 大师 | ≥ 2000 |

徽章等级随积分自动升级，展示在用户资料和 Agent 卡片上。

## 6. 国际化策略

- **主语言**：简体中文（`zh-CN`）
- **辅语言**：英文（`en`）
- 所有 UI 文案通过 `i18next` 管理，新增功能必须同时补充两种语言键值。
- Agent 数据模型支持 `nameZh/nameEn`、`descriptionZh/descriptionEn` 等双语字段。

---

## 7. 非功能性需求

### 7.1 性能

- 广场列表页首屏加载 < 1s（20 条数据 + 分页）
- 搜索响应 < 500ms（PostgreSQL GIN 索引 + tsvector）
- 排名重计算异步执行，不影响用户请求

### 7.2 安全

- 密码使用 bcryptjs（salt rounds 12）哈希存储
- JWT Access Token 15 分钟过期，Refresh Token 7 天过期
- Fastify 速率限制：100 req / 1 minute
- SQL 查询全部使用参数化占位符

### 7.3 可扩展性

- 使用 pnpm workspace 模块化组织代码
- 共享类型与常量抽离到 `packages/shared`
- 后端路由按模块拆分，便于后续增加新功能

## 9. 术语表

| 术语 | 解释 |
|------|------|
| Agent | 智能体，指具有特定系统提示词和可选工具配置的 AI 角色 |
| MCP | Model Context Protocol，Anthropic 提出的工具扩展协议 |
| OpenClaw | 一个开源的多 Agent 协作框架 |
| Hire | 在 HireAgent 中指"使用/雇佣"某个 Agent 配置，表现为导出或复制 |
| Ranking Score | 综合排名分，决定 Agent 在广场中的默认排序 |
| Quality Score | 质量分，根据 Agent 信息完整度自动计算 |
